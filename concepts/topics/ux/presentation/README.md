# Overview

Presentation choices determine what people notice first and how hard the
interface feels to read.

# Content Structure

- Give larger chunks of text short headings
- Think about what people need to know to complete the task
- Explain domain concepts in plain language before expecting action

# Color

- Dark themes are useful in low-light environments, but should be chosen based
  on context rather than fashion
- Black, white, and neutral tones make good anchors when bright color would
  compete with the task
- Overly saturated color can overwhelm users and distract from their goals
- Off-white or gray surfaces can make light text feel less harsh than pure black
  and white contrast in some interfaces

# Shape

- Strictly vertical and rectangular layouts are not the only option
- Non-orthogonal shapes can create emphasis, but should still preserve clarity
- Reference: [Zod v4](https://v4.zod.dev/v4)
